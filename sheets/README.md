# sheets/ — Google Apps Script Web App

Приёмник данных от serverless-функции (`yandex-function/`): складывает незакреплённые заявки и метрики
эксперимента в Google Sheets. Формат запроса/ответа — см. `CONTRACT.md`, раздел 4.

## 1. Создать Google Sheet

1. Откройте [sheets.google.com](https://sheets.google.com) → создайте новую таблицу
   (например, «Заявки на обучение — эксперимент»).
2. Листы «Незакреплённые заявки» и «Метрики эксперимента» создавать вручную
   **не нужно** — `Code.gs` сам создаёт их при первой записи, с правильными
   заголовками столбцов. Если хочете создать заранее — назовите листы точно
   так же (с заглавными буквами и «ё»), иначе скрипт создаст дублирующий лист.

## 2. Вставить код

1. В таблице: **Расширения (Extensions) → Apps Script**.
2. Откроется редактор с файлом `Code.gs` (или `Код.gs`) — удалите содержимое
   по умолчанию и вставьте весь текст из `sheets/Code.gs` этого репозитория.
3. Сохраните проект (значок дискеты или `Ctrl+S`). Название проекта можно
   оставить любым.

## 3. Задать секрет (SHARED_SECRET)

Секрет проверяет, что запрос пришёл от нашей serverless-функции, а не от
случайного вызова по URL. Задаётся **не в коде**, а через настройки проекта:

1. В редакторе Apps Script слева — значок шестерёнки **Project Settings**
   (Настройки проекта).
2. Прокрутите до раздела **Script Properties** (Свойства скрипта).
3. Нажмите **Add script property**.
4. Property: `SHARED_SECRET`. Value: любая длинная случайная строка
   (например, сгенерируйте её командой `openssl rand -hex 32` или любым
   генератором паролей).
5. **Save script properties**.

Эта же строка пойдёт в переменную окружения функции `SHEETS_WEBHOOK_SECRET`
(см. ниже).

## 4. Развернуть как Web App

1. В редакторе Apps Script — кнопка **Deploy → New deployment**.
2. Если предложит выбрать тип — иконка шестерёнки рядом с «Select type» →
   **Web app**.
3. Настройки деплоя:
   - **Execute as**: `Me` (ваш аккаунт — так у скрипта будет доступ на запись
     в таблицу без OAuth-запроса к вызывающей стороне).
   - **Who has access**: `Anyone` (в старых интерфейсах — «Anyone with the
     link» / «Все, у кого есть ссылка»). Это обязательно, иначе serverless-функция
     не сможет вызвать скрипт без входа в Google-аккаунт.
4. **Deploy**. При первом деплое Google попросит авторизовать доступ скрипта
   к вашей таблице — пройдите стандартный flow (может показать предупреждение
   «Google hasn't verified this app» — это ожидаемо для собственного скрипта,
   жмите Advanced → Go to (project name)).
5. После деплоя откроется окно с **Web app URL** — он выглядит как
   `https://script.google.com/macros/s/XXXXXXXX/exec`. Скопируйте его.

## 5. Передать URL и секрет в yandex-function

Значения, полученные выше, — это переменные окружения serverless-функции (см.
`CONTRACT.md`, раздел 5):

| Значение из этого документа | Переменная окружения функции |
|---|---|
| Web app URL (шаг 4.5)        | `SHEETS_WEBHOOK_URL` |
| SHARED_SECRET (шаг 3.4)      | `SHEETS_WEBHOOK_SECRET` |

Задаются в консоли Yandex Cloud, на странице функции (см. `yandex-function/README.md`, шаг 3 «Задать переменные окружения»).

## 6. Обновление кода после изменений

Если правите `Code.gs` — изменения в редакторе Apps Script применяются к
Web App только после **нового** деплоя (или после Deploy → Manage deployments
→ Edit → New version для того же URL, чтобы URL не менялся). Простое
сохранение файла (`Ctrl+S`) обновляет только черновик, но не действующий
Web App.

## Нюансы Apps Script, о которых стоит знать

- **doPost всегда возвращает HTTP 200.** Apps Script Web Apps не умеют отдавать
  4xx/5xx — все ошибки (неверный secret, невалидный JSON, неизвестный `sheet`)
  возвращаются как `{ok:false, error:'...'}` в теле с кодом 200. Функция должна
  проверять поле `ok`, а не HTTP-статус.
- **Конкурентная запись.** Если несколько заявок отправляются почти
  одновременно, `SpreadsheetApp.appendRow` в редких случаях может гонки при
  определении последней строки. `Code.gs` оборачивает запись в
  `LockService.getScriptLock()` — это последовательно выравнивает конкурентные
  вызовы `doPost` в рамках одного Apps Script проекта. Для объёма теста
  (десятки заявок) этого достаточно; таймаут ожидания лока — 30 секунд.
- **Квоты.** Бесплатный Google-аккаунт: лимит ~20 000 вызовов URL Fetch/day
  не относится к doPost напрямую, но есть общий лимит на время выполнения
  скрипта (6 минут на один вызов) и на число триггеров/скриптов в день —
  для объёма прототипа (десятки-сотни заявок) более чем достаточно.
- **Авторизация исполнения.** Execute as "Me" означает, что скрипт пишет в
  таблицу от имени владельца, даже если вызывающая сторона (serverless-функция)
  не аутентифицирована в Google — именно поэтому единственная защита
  от посторонних вызовов — shared secret в теле запроса. Никогда не
  публикуйте Web App URL в клиентском (браузерном) коде — он должен быть
  известен только функции (переменная окружения на сервере).
- **Пересоздание деплоя меняет URL.** Если вы удалите деплоймент и создадите
  новый (а не «New version» существующего), URL изменится — придётся заново
  обновлять `SHEETS_WEBHOOK_URL` в переменных окружения функции.
