# Прототип веб-формы заявки на обучение

Статус: прототип для эксперимента (см. `бриф_прототип_веб_формы_заявки.md`). Реальные интеграции частично подключены — см. таблицу ниже.

Схема данных между частями — единый источник правды: **[`CONTRACT.md`](./CONTRACT.md)**. Если что-то меняете в полях/эндпоинтах — правьте контракт первым.

## Состав

| Папка | Что это | Куда деплоится |
|---|---|---|
| `site/` | Статичная форма (HTML/CSS/JS, без сборки) | GitHub Pages |
| `yandex-function/` | Serverless-прокси в Б24/Sheets, генерация xlsx | Yandex Cloud Functions |
| `scripts/` | Разовый Node-скрипт экспорта каталога курсов из Б24 | запускается локально вручную |
| `sheets/` | Google Apps Script — приёмник в Google Sheets | Google Apps Script Web App |

Изначально serverless-часть была на Cloudflare Workers — заменена на Yandex Cloud Functions, так как Cloudflare нестабильно работает в РФ без VPN, а заказчику VPN не нужен.

## Статус по шагам деплоя

1. **Репозиторий и GitHub Pages** — репозиторий создан и запушен (`tashakibanova/application_for_training`), настроен workflow `.github/workflows/pages.yml` (публикует `site/`). Ждём включения Pages в Settings → Pages → Source → GitHub Actions на стороне владельца репозитория.
2. **DaData** — ключ получен и вписан в `site/app.js`, проверен живым запросом к API.
3. **Каталог курсов** — реальный каталог выгружен: `site/courses.json` содержит **798 курсов** из Б24 (iblockId 21, разделы ЦДО КПК/ПП в продаже, ЕРЛ Сколково/Сколково: ПП, ЕРЛ Логопедия/дефектология, ЕРЛ Охрана труда — подробности в `scripts/README.md` и `CONTRACT.md` §6). Форма использует поиск по ключевым словам вместо выпадающего списка (см. `site/app.js`, функция `wireListenerCourseSearch`) — обычный `<select>` неюзабелен при таком объёме.
4. **Yandex Cloud Function** — код готов (`yandex-function/`), проверен локально (валидация, генерация xlsx). Деплой в облако ещё не выполнялся — инструкция в `yandex-function/README.md`.
5. **Bitrix24 webhook** — получен, права `crm`+`catalog` проверены живыми запросами (профиль, каталог). Пока не прописан в `yandex-function` (нет задеплоенной функции).
6. **Google Sheets / Apps Script** — код готов (`sheets/Code.gs`), не задеплоен.

## Что осталось сделать

1. Включить GitHub Pages (Settings → Pages → Source → GitHub Actions) в репозитории и перезапустить упавший workflow, если понадобится (см. `Settings → Environments → github-pages → Deployment branches` — вебхук/среда должны разрешать ветку `master`).
2. Задеплоить `sheets/Code.gs` как Web App — инструкция `sheets/README.md`. Получить `SHEETS_WEBHOOK_URL` и придумать `SHEETS_WEBHOOK_SECRET`.
3. Задеплоить `yandex-function/` — инструкция `yandex-function/README.md`. Прописать туда `BITRIX_WEBHOOK_URL`, `SHEETS_WEBHOOK_URL`, `SHEETS_WEBHOOK_SECRET`, `ALLOWED_ORIGIN`.
4. Вписать адрес задеплоенной функции в `site/app.js` (`FUNCTION_BASE_URL`), закоммитить и запушить — GitHub Actions передеплоит форму автоматически.
5. Сквозной прогон: отправить тестовую заявку по ссылке `https://.../?deal=<реальный ID сделки>`, проверить xlsx-вложение в комментарии сделки и кириллицу в нём.
6. Ссылки менеджерам — вручную добавлять `?deal=<ID сделки>` к адресу формы.

## Как решены открытые вопросы брифа (для прототипа, не окончательно)

1. **Заявка без `?deal=`** — пишется в лист «Незакреплённые заявки» с пометкой, разбирается менеджером вручную. Дополнительно функция подстраховывается: если `dealId` указан, но вызов Б24 упал по любой причине, заявка тоже падает в этот лист (а не теряется) — с пояснением ошибки в поле `note`.
2. **Незавершённое заполнение** — реализовано технически (минимально): `sendBeacon` на `beforeunload`/`pagehide`/скрытие вкладки шлёт `startedAt` и число тронутых строк слушателей в лист «Метрики эксперимента» со статусом `abandoned`.
3. **Частота обновления `courses.json`** — ручной запуск `scripts/export-catalog.js`; периодичность определяется по факту того, как часто меняется каталог во время теста.

## Известные отклонения от точной буквы CONTRACT.md (обоснованные)

- Формат вложения файла в `crm.timeline.comment.add` — используется `FILES: [[fileName, base64]]` (проверено по официальной документации Б24 REST).
- В конверт `sheet: "unbound"` добавлено необязательное поле `xlsxBase64` сверх минимального набора из CONTRACT.md §4.1 — чтобы данные по неприкреплённым заявкам не терялись.
- Роутинг `yandex-function` — через query-параметр `?action=submit`/`?action=track`, а не через путь, как в изначальной Cloudflare-версии (прямой URL вызова функции не поддерживает под-пути без API Gateway).
