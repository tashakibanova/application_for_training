# yandex-function — как опубликовать через браузер (без терминала)

Это серверная часть формы заявки: она принимает данные из формы (папка `site/`),
делает из них файл `.xlsx` и отправляет либо комментарием в сделку Битрикс24,
либо строкой в Google-таблицу (если сделки нет — см. `sheets/README.md`).
Код уже написан и лежит в этой папке, в файлах `index.js` и `package.json` —
их трогать не нужно, нужно только «загрузить» их в Yandex Cloud через сайт
консоли управления, без установки чего-либо на компьютер.

Понадобится: аккаунт Yandex Cloud (см. шаг 0) и открытые рядом файлы
`index.js` и `package.json` из этой папки — их содержимое будете копировать
в консоль.

Везде ниже «консоль» = сайт **console.yandex.cloud**, открытый в обычном
браузере. Никакой установки программ и никакого «чёрного окна с текстом»
(терминала) не требуется.

> Названия кнопок в интерфейсе Yandex Cloud время от времени немного меняются
> (переставляют слова, меняют регистр). Если какая-то кнопка называется чуть
> иначе, чем написано ниже — ищите рядом по смыслу, обычно нужная кнопка
> синяя и находится там же, где описано.

## Шаг 0. Аккаунт и оплата

Зайдите на [console.yandex.cloud](https://console.yandex.cloud) под своим
Яндекс ID (обычной яндекс-почтой) — консоль сама предложит создать облако и
каталог (folder), если их ещё нет. Чтобы пользоваться сервисами (даже в
пределах бесплатного стартового гранта), Yandex Cloud попросит один раз
создать **платёжный аккаунт** и привязать банковскую карту (Мир, Visa или
MasterCard) — это стандартное требование облачных провайдеров, деньги с
карты без вашего ведома не спишутся, пока вы не переключитесь на платное
потребление сверх гранта. Если консоль уже открывается и просит выбрать
каталог — этот шаг уже пройден, можно переходить к шагу 1.

## Шаг 1. Создать функцию

1. В консоли слева (или через строку поиска сервисов вверху) найдите и
   откройте сервис **Cloud Functions**.
2. Проверьте, что вверху страницы выбран правильный каталог (folder) — если
   каталог только один, он выбран автоматически.
3. Нажмите кнопку **Создать функцию**.
4. В поле имени введите, например, `zayavka-function` (только латинские
   строчные буквы, цифры и дефисы, начинается с буквы).
5. Нажмите **Создать**.

Функция создана, но пока пустая — код в неё нужно загрузить отдельным шагом
(это называется «создать версию функции»).

## Шаг 2. Загрузить код (создать версию функции)

1. На странице функции найдите раздел **Последняя версия** и нажмите
   **Создать в редакторе**.
2. В списке сред выполнения выберите **Node.js** самой новой доступной
   версии (подходит любая версия 18 и новее — если в списке есть, например,
   `Node.js 22`, берите её).
3. Отключите переключатель **Добавить файлы с примерами кода** (он подставляет
   демо-код, который нам не нужен).
4. Нажмите **Продолжить**.
5. Откроется редактор кода. Он покажет один файл (обычно `index.js`) слева
   в списке файлов.
   - Откройте на своём компьютере файл `yandex-function/index.js`, выделите
     весь текст, скопируйте и вставьте в редактор вместо содержимого-заглушки.
   - Рядом со списком файлов слева найдите кнопку для создания нового файла
     (обычно это «+» или подпись «Создать файл» — точная подпись может
     немного отличаться). Создайте файл с именем **`package.json`** (именно
     так, с этим расширением) и вставьте в него весь текст из
     `yandex-function/package.json` этого проекта.
   - Это важно: без файла `package.json` Yandex Cloud не узнает, что функции
     нужна библиотека `xlsx`, и код упадёт с ошибкой при первом запуске.
6. В поле **Точка входа** впишите:
   ```
   index.handler
   ```
   (это означает: «взять файл `index.js`, а в нём — функцию `handler`»,
   именно так экспортирован код).
7. Ниже задайте параметры версии:
   - **Таймаут** — поставьте `15` секунд (запрос ходит во внешние сервисы —
     Битрикс24 и Google, стандартных 3 секунд может не хватить).
   - **Память** — можно оставить `128 МБ` по умолчанию; если позже увидите
     в логах ошибку про нехватку памяти, увеличьте до `256 МБ`.
   - **Сервисный аккаунт** — можно оставить не выбранным, он не нужен.
8. Тут же на этой странице (или в отдельном блоке **Переменные окружения**)
   впишите 4 переменные — как это сделать, подробно в шаге 3 ниже. Можно
   сделать это сразу, а можно — уже после того, как функция появится
   (тогда просто вернитесь на эту же страницу ещё раз, см. шаг 3).
9. Нажмите **Сохранить изменения**.

Yandex Cloud соберёт версию функции и **сам установит зависимость `xlsx`**
по данным из `package.json` — ничего архивировать и загружать отдельно не
нужно. Сборка может занять минуту-две.

### Если через редактор не получилось (запасной способ — ZIP-архив)

Если по какой-то причине автоматическая установка `xlsx` не сработала (в
логах функции будет ошибка вроде `Cannot find module 'xlsx'`), можно
загрузить код вместе с уже установленными зависимостями:

1. На компьютере, в папке `yandex-function/`, нужно один раз локально
   выполнить установку зависимостей (это единственное место, где без
   помощи разработчика не обойтись — либо попросите кого-то один раз
   выполнить `npm install` в этой папке, либо используйте готовую папку
   `node_modules`, если она уже была подготовлена для вас).
2. Заархивируйте в ZIP содержимое папки `yandex-function/` **целиком**,
   включая `index.js`, `package.json` и папку `node_modules` (сам ZIP-файл
   должен содержать эти файлы/папки прямо внутри, без лишней папки-обёртки
   сверху).
3. На шаге 2.4 выше, при подготовке кода, в поле **Источник кода** выберите
   **ZIP-архив** вместо редактора и прикрепите получившийся файл.
4. Учтите ограничение: через консоль напрямую можно загрузить архив весом
   **до 3,5 МБ**. Архив с `node_modules` для `xlsx` обычно получается
   больше (порядка 4–5 МБ), поэтому впрямую прикрепить его в форме может не
   получиться — консоль в этом случае предложит сначала загрузить ZIP в
   **Object Storage** (это тоже делается мышкой через браузер, без
   терминала — консоль подскажет кнопку перехода) и указать функции ссылку
   на файл в хранилище.

Из-за этого ограничения **первым делом всегда пробуйте способ через
редактор (вставка кода в двух файлах) из основного шага 2** — он проще и
не упирается в лимит размера.

## Шаг 3. Задать переменные окружения

Функции нужно передать 4 секретных значения — они не должны быть видны в
самом коде. Если вы не заполнили их на шаге 2.8, сделайте так:

1. Откройте функцию → вкладку **Редактор**.
2. Найдите блок **Параметры** (или **Переменные окружения**).
3. Для каждой строки из таблицы ниже: впишите имя переменной, впишите
   значение и нажмите **Добавить** (кнопка добавления переменной — так
   добавляются все 4 по очереди).
4. Когда впишете все 4 — нажмите **Сохранить изменения** внизу. Это создаст
   новую версию функции с этими переменными (это нормально, старая версия
   просто станет неактуальной).

| Переменная | Что вписать | Где взять значение |
|---|---|---|
| `BITRIX_WEBHOOK_URL` | Ссылка входящего вебхука Битрикс24, вида `https://ваш-портал.bitrix24.ru/rest/1/xxxxxxxx/` | В настройках вашего портала Битрикс24, там где создавался вебхук для этого проекта |
| `SHEETS_WEBHOOK_URL` | Ссылка Web App, вида `https://script.google.com/macros/s/XXXXXXXX/exec` | `sheets/README.md`, шаг 4.5 |
| `SHEETS_WEBHOOK_SECRET` | Секретная строка (пароль) | `sheets/README.md`, шаг 3.4 — то же самое значение, что вписано в Script Properties → `SHARED_SECRET` |
| `ALLOWED_ORIGIN` | Адрес сайта с формой, например `https://ваш-логин.github.io` | Адрес, по которому опубликована форма (GitHub Pages) |

Никогда не вписывайте эти значения прямо в файл `index.js` — только сюда,
в переменные окружения.

## Шаг 4. Разрешить вызов без входа в аккаунт (публичный доступ)

Форма на сайте — это просто браузер обычного посетителя, у него нет и не
может быть «пропуска» (авторизации) в ваш Yandex Cloud. Поэтому нужно явно
разрешить вызывать функцию всем:

1. Откройте функцию → вкладку **Обзор**.
2. Включите переключатель **Публичная функция** (в английской версии
   интерфейса — «Public function» / «Allow unauthenticated invoke»; если
   переключатель называется чуть иначе — ищите на этой же странице
   формулировку про вызов без авторизации).

Без этого шага форма на сайте будет получать ошибку доступа при отправке
заявки.

## Шаг 5. Получить ссылку (URL) функции и вписать в форму

1. На той же вкладке **Обзор** страницы функции есть поле с готовой ссылкой
   на функцию — она выглядит так:
   ```
   https://functions.yandexcloud.net/d4e...
   ```
   (длинный идентификатор после последнего слэша — это ID именно вашей
   функции). Скопируйте её целиком.
2. Откройте файл `site/app.js` в этом проекте и найдите строку:
   ```js
   const FUNCTION_BASE_URL = 'https://REPLACE-ME.functions.yandexcloud.net';
   ```
3. Замените значение на скопированную ссылку, например:
   ```js
   const FUNCTION_BASE_URL = 'https://functions.yandexcloud.net/d4e...';
   ```
4. Сохраните файл и опубликуйте сайт как обычно (см. корневой README /
   `.github/workflows/pages.yml`) — форма сама допишет к этому адресу
   `?action=submit` или `?action=track`, где нужно.

## Шаг 6. Проверить, что функция отвечает

Без терминала полноценный POST-запрос с телефона или ноутбука отправить
проще всего одним из способов:

- **Самый простой способ** — просто вставить готовую ссылку из шага 5
  (без `?action=...`) с добавлением `?action=submit` в адресную строку
  браузера и открыть её. Браузер сделает GET-запрос (форма отправляет
  POST, так что это не «настоящая» проверка отправки), но по ответу можно
  сразу понять, всё ли настроено:
  - Пришёл JSON вида `{"ok":false,"error":"not_found"}` — отлично, функция
    развёрнута и доступна публично, просто GET-запрос не поддерживается
    (это ожидаемо).
  - Ошибка вида «Доступ запрещён» / `403 Forbidden` / `Unauthorized` — не
    выполнен шаг 4 (публичный доступ).
  - Ошибка вида «страница не найдена» на уровне самого браузера/домена —
    не тот URL, перепроверьте шаг 5.
- **Полноценная проверка POST-запроса** — либо просто откройте реальную
  форму сайта (после того как вписали ссылку по шагу 5) и отправьте тестовую
  заявку — это самый надёжный способ проверить всё целиком; либо
  воспользуйтесь браузерным расширением для отправки запросов без
  терминала, например **Talend API Tester** или **Requestly** (ищутся в
  Chrome Web Store), указав в них: метод `POST`, адрес — ссылка из шага 5 с
  `?action=submit` на конце, тело запроса — `{}`. Ожидаемый ответ —
  `400` с текстом вроде `{"ok":false,"error":"missing_organization"}` —
  это означает, что функция получает запросы и проверяет данные (пустая
  заявка закономерно отклонена, но сама функция работает).

## Шаг 7. Обновить код после правок

Если позже в `index.js` что-то поменяется, нужно загрузить новую версию —
старая при этом не портится, просто перестаёт быть «последней»:

1. Откройте функцию в консоли → раздел **Последняя версия** → снова
   **Создать в редакторе** (или аналогичная кнопка создания версии).
2. Повторите шаг 2: вставьте обновлённый текст `index.js` (и, если он
   менялся, `package.json`), впишите точку входа `index.handler`, при
   необходимости — таймаут и память.
3. Переменные окружения нужно указать заново (они не переносятся
   автоматически между версиями) — используйте те же значения, что и в
   шаге 3.
4. Сохраните — новая версия станет действующей, ссылка на функцию (URL) при
   этом **не меняется**, во `site/app.js` ничего трогать не нужно.
5. Публичный доступ (шаг 4) тоже, как правило, нужно перепроверить/включить
   заново для новой версии, если он вдруг оказался выключен.

## Если что-то не работает

- **Форма получает ошибку доступа / 403 / Unauthorized.** Не включён
  публичный вызов — вернитесь к шагу 4 и убедитесь, что переключатель
  «Публичная функция» включён именно для последней (действующей) версии.
- **В ответе `xlsx_generation_failed` или в логах функции ошибка
  `Cannot find module 'xlsx'`.** Зависимость не подтянулась при сборке —
  либо повторите шаг 2 внимательнее (файл `package.json` должен быть
  создан отдельным файлом рядом с `index.js`, с точным именем и валидным
  содержимым), либо используйте запасной способ с ZIP-архивом и
  `node_modules`, описанный в конце шага 2.
- **Заявка не долетает ни в Битрикс24, ни в Google-таблицу.** Проверьте
  переменные окружения (шаг 3) — особенно что `SHEETS_WEBHOOK_SECRET` в
  функции совпадает один-в-один с `SHARED_SECRET`, заданным в Google Apps
  Script (`sheets/README.md`, шаг 3.4), и что `SHEETS_WEBHOOK_URL` — это
  именно последний задеплоенный Web App URL, а не старый (URL меняется,
  если в Apps Script пересоздать деплоймент, а не обновить существующий —
  см. `sheets/README.md`, раздел «Нюансы Apps Script»).
- **Логи функции.** У каждой функции в консоли есть вкладка с логами
  (обычно называется **Логи** — рядом с «Обзор» и «Редактор») — туда
  попадают все `console.error(...)` из кода, это самый быстрый способ
  понять, что именно пошло не так при конкретном вызове.
